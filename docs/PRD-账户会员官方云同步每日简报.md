# PRD：账户、会员、官方云同步、每日简报

> 本文件来自客户端阶段的原始 PRD，其中“仓库内不实现”指客户端仓库 `WearOS_ClassingTimeTable`。后端功能现已在当前 `classing_backend` 仓库实现。

## 1. 目标
- 为 `mobile` 增加完整账号体系入口：登录、注册、退出、密码重置、会员状态、兑换码。
- 为 `mobile` 增加官方云同步 `OFFICIAL`，固定基址 `https://api-classing.underflo.ink`，且仅会员可用。
- 将移动端第二主页面正式定义为 `Dashboard`，统一承接今日概览与下一节课视图。
- 为 `mobile` 增加每日简报设置，支持系统通知、邮件、双通道。
- `wear` 只展示账号与云同步的只读摘要，不输入账号、不持有令牌、不直连账号接口。

## 2. 范围

### 2.1 客户端内实现
- `mobile` 设置页新增 `Account`、`Daily Briefing`。
- `mobile` 云同步设置新增 `OFFICIAL`、同步项目、官方频率。
- `mobile` Dashboard 新增：
  - 今日总课数
  - 今日剩余课时
  - 下一节课
  - 当前状态
  - 热力图
- `wear` 设置页新增 About 入口。
- `wear` 关于页新增版本与 `ICP 备案号：待补充`。
- `wear` 云同步页/设置页展示：
  - 是否已登录
  - 是否会员
  - 当前云同步方式
  - 官方云同步是否可用

### 2.2 仓库内不实现
- 不实现任何后端服务代码。
- 不实现兑换码后台管理界面。
- 不实现邮件实际发送集群。

## 3. 角色与设备分工
- 手机端：主控端，持有账号令牌，负责账号接口调用、会员状态刷新、官方云同步、每日简报本地通知调度。
- 手表端：只读端，只消费手机下发的状态摘要与课程/设置快照。
- 后端：负责鉴权、会员、兑换码、密码重置、官方云同步数据存储、每日简报邮件订阅与投递。

## 4. 核心规则
- 登录标识为 `identifier`，支持邮箱或用户名登录。
- 注册字段必须包含：`username`、`email`、`password`。
- 密码重置始终走邮箱。
- `OFFICIAL` 云同步固定基址，不支持修改。
- 任意时刻仅允许启用一种云同步 Provider。
- 同步项目仅限：
  - `TIMETABLE`
  - `MOBILE_SETTINGS`
  - `WEAR_SETTINGS`
- 账号令牌、会员缓存、WebDAV 密码、Drive Token 不进入任何云同步文档。
- 官方云同步仅会员可用。
- 兑换成功后客户端需立即刷新会员状态并尝试解锁官方云同步。

## 5. 主要流程

### 5.1 登录/注册
1. 用户在 `Account` 页面提交登录或注册。
2. 客户端保存 `accessToken` 与 `refreshToken`。
3. 客户端立即请求 `/account/me` 与 `/membership/status`。
4. 客户端更新本地账号摘要、会员摘要、Wear 只读快照。

### 5.2 兑换码升级会员
1. 用户输入兑换码。
2. 客户端调用 `POST /api/v1/membership/redeem`。
3. 后端原子完成校验、扣减、升级、审计。
4. 客户端刷新 `/account/me`，解锁 `OFFICIAL` Provider。

### 5.3 官方云同步
1. 用户在 `Cloud Sync` 选择 `OFFICIAL`。
2. 若未登录或未开会员，展示锁定态与升级入口。
3. 若已满足条件，允许保存、测试连接、手动同步、自动频率选择。
4. 定时同步频率受 `officialSyncFrequency` 控制。

### 5.4 每日简报
1. 用户设置 `enabled`、`channel`、`time`。
2. `APP_NOTIFICATION` 由客户端本地调度。
3. `EMAIL` 或 `BOTH` 时客户端将配置提交后端。
4. 后端负责邮件订阅、任务调度、SMTP 投递与邮箱池切换。

## 6. 验收标准
- 非会员选择官方云同步时看到锁定态，兑换成功后无需重启即可解锁。
- Dashboard 成为下一节课唯一主展示位置，课表页不再出现对应 Hero。
- 手表可只读看到登录/会员/云同步摘要。
- mobile 和 wear 关于页均显示 `ICP 备案号：待补充`。
