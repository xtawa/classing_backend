# 实施清单：移动端与 Wear 端改造

> 本文件记录客户端改造边界。当前后端仓库已经实现服务端、SMTP 任务框架与管理台；客户端仓库仍保持只交付客户端代码和契约文档。

## 已完成方向
- `mobile`
  - 新增账号/会员数据模型与本地持久化字段。
  - 新增 `AuthCredentialStore`。
  - 新增账号接口客户端。
  - 新增官方云同步 Provider。
  - 新增同步项目 `SyncScope`。
  - 新增官方频率 `OfficialSyncFrequency`。
  - 新增每日简报配置与本地通知调度。
  - Dashboard 承接下一节课与今日统计。
  - 设置页新增 `Account`、`Daily Briefing` 入口。
  - 关于页新增 `ICP 备案号：待补充`。
- `wear`
  - 新增 About 页面与导航。
  - 新增手机下发云状态摘要的接收与展示。
  - 设置页增加只读账号摘要卡。
  - 云同步页增加只读登录/会员/Provider/官方可用性展示。

## 客户端仍需联合验证
- 登录、注册、登出与后端返回字段是否完全一致。
- 密码重置邮件 token 格式。
- 兑换成功后的会员状态刷新是否满足服务端最终实现。
- 官方云同步接口路径、ETag、幂等键规则。
- 每日简报邮件订阅接口与测试发送接口。

## 测试建议
- mobile：
  - 登录后重启应用，状态仍保留。
  - 退出登录后官方云同步立即锁定。
  - 切换 `syncScopes` 后仅同步选中域。
  - `OFFICIAL` 频率切换后周期任务更新。
  - 每日简报时间到点触发本地通知。
- wear：
  - 收到手机状态快照后 About/Settings/Cloud 页面展示正确。
  - 不登录时只显示只读锁定态。

- 本仓库只交付客户端入口、状态与文档。
- 后端实现、SMTP 基础设施、管理台不在本次提交内。
