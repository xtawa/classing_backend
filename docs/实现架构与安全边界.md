# Classing 后端实现架构与安全边界

## 1. 运行结构

单个 Go 进程包含：

- REST API
- Material You 管理台静态资源
- 自动数据库迁移
- 每日简报时区调度器
- SMTP 邮箱池投递工作器
- 健康检查和 JSON 日志

生产环境推荐 PostgreSQL。SQLite 使用纯 Go 驱动，作为单机和测试模式。两种数据库共用同一套业务表与查询。

## 2. 权限

### ADMIN

- 读取和更新全部用户角色、状态。
- 读取和删除全部课表项目。
- 生成、查询、吊销兑换码。
- 授予或撤销会员。
- 管理 SMTP 邮箱池和简报任务。
- 查看审计日志并修改允许的运行时设置。

### USER

- 管理自己的资料、邮箱和密码。
- 管理自己的课表项目。
- 查看会员和兑换权益。
- 使用会员专属官方云同步。
- 管理自己的每日简报。

Wear 默认继续消费手机下发的只读摘要；自动状态不可用时可经短时二维码建立独立加密会话。二维码不含 token 或轮询密钥，Wear token 不进入 Data Layer 或云文档。

## 3. 认证

- access token：HMAC-SHA256，默认 15 分钟。
- refresh token：32 字节随机不透明 token，数据库仅保存 SHA-256 hash。
- refresh token 每次刷新后立即轮换；同一 token、IP 与 User-Agent 仅在 5 秒并发宽限内重放同一 replacement session，失败结果不缓存。
- 密码使用 bcrypt cost 12。
- 修改密码、管理员调整角色或停用账户时提升 `auth_epoch`，现存 access token 立即失效。
- 密码变更与重置同时撤销全部 refresh token。
- 密码重置 token 一次性使用，并记录申请 IP 和 User-Agent。

## 4. 官方云同步

- 固定 Provider：`OFFICIAL`。
- 登录账户可同步 mobile/Wear 设置；仅有效会员可额外读写课表域。
- GET 返回带引号 `ETag`。
- PUT 支持 `If-Match`；过期版本返回 412。
- 兼容现有客户端未发送 `Idempotency-Key` 的行为；发送时会缓存请求 hash 和响应。
- 默认最大文档 2 MiB。
- 仅允许合法 JSON 文档。

## 5. 简报与邮箱池

- 邮箱密码不写入数据库；只保存 `env:VARIABLE_NAME` Secret 引用。
- 调度按每个用户的 IANA 时区计算。
- `(user_id, target_date, channel)` 唯一约束避免重复任务。
- 邮箱按当天使用量最低优先，并受每日额度限制。
- 587 自动协商 STARTTLS；465 使用隐式 TLS；最低 TLS 1.2。
- 失败任务指数式延后，超过限制转为 FAILED。

## 6. 审计

记录：

- 账户注册、登录、退出、密码变化。
- 用户资料、角色和状态变化。
- 会员兑换、授予与撤销。
- 课表创建、更新与删除。
- 官方云写入版本和大小。
- 邮箱池、任务重试和系统设置变化。

日志包含 request ID、actor、target、IP、User-Agent 和受控 metadata，不记录密码、JWT 或 SMTP Secret 明文。

## 7. 当前边界

- 管理台会话保存在浏览器 `sessionStorage`，关闭标签页后清除；页面不加载第三方脚本，并由 CSP 限制为同源。
- access token 为无状态 token，但 `auth_epoch` 可以立即撤销账户级会话。
- PostgreSQL 高可用、对象存储备份和外部 Secret Manager 属于部署基础设施，需要由运行环境提供。
- 邮件模板目前为安全的纯文本基础模板，可在后续接入品牌化 HTML 模板和课程内容聚合器。
